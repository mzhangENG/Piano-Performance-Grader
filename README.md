# Piano-Performance-Grader
This was the first ever personal project I did in college, specifically the winter break of my freshmen year. After taking my first ever coding class (CS 159 - Intro to C Programming), I became really interested in programming and decided to spend my winter breaking learning python and build something that I thought was cool. I've played piano since the 4th grade, and I wanted to combine my newly gained interest in coding with this lifelong passion of mine. 

In general, this application is meant to score the note and tempo accuracy of a piano performance by comparing it to an ideal reference. It uses Librosa to extract dominant frequency data at onset frames, allowing the program to build a list of the played notes in the order in which they were played. This is done for both the ideal reference and user performance, then both lists are compared via a Levenshtein Distance Algorithm to generate a final score grading. Tempo is scored by comparing the spacing between detected note onsets in the reference and test recordings, allowing small timing differences, and then converting how closely those spacing patterns match into a percentage.
